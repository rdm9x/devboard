# Devboard — Гайд для Owner'а (v3.0)

> **Что это?** Дашборд на localhost:4999 для управления командой AI-агентов, организованной в отделы.
>
> **Кто использует?** Ты (owner) и команда лидов отделов.
>
> **Сколько времени на освоение?** 15 минут основ + 5 минут на каждую новую функцию.

---

## Структура команды (Управляющий + 11 отделов)

После v3.0 команда организована как **настоящая компания**:

```
                    ТЫ (Owner)
                       ↕ (общий чат в дашборде)
                 Управляющий (ассистент)
                       ↕ (раздаёт работу)
    ┌──────────────────┼──────────────────┐
    ↓                  ↓                  ↓
 Маркетинг          Разработка          Юристы
 (лид + team)       (лид + team)        (лид + team)
 + копирайтер      + бэкенд             + юрист
 + дизайнер        + frontend           + специалист
 + аналитик        + qa                 + ...
                   + devops
```

**Управляющий** — это твой персональный ассистент с долгосрочной памятью. Он:
- Помнит стиль и предпочтения
- Знает что делают все отделы
- Раздаёт работу лидам
- Координирует межотдельные проекты через **планёрку**

## Как работать (типичный день)

### 1️⃣ Создание задачи

**Способ A: Через дашборд (рекомендуется)**
1. Открыть http://localhost:4999
2. Нажать **«+ Новая задача»**
3. Заполнить форму:
   - **Что делать** (обязательно): понятное описание
   - **Для кого** (опционально): выбрать отдел, если уже знаешь (маркетинг, разработка, юристы)
   - **Приоритет** (опционально): обычные задачи = P2
4. Нажать **«Создать»**

**Что произойдёт:**
- Если отдел выбран → задача сразу в очередь лида этого отдела
- Если отдел не выбран → задача идёт Управляющему (в его inbox)
- Управляющий декомпозирует, опрашивает лидов (если нужно), распределяет

### 2️⃣ Запуск сессии

**Когда сессия нужна:**
- Ты добавил задачу, и хочешь чтобы команда начала работать
- Хочешь проверить статус (кто что делает в реальном времени)

**Как запустить:**
1. В дашборде нажать **«▶ Запустить команду»** (кнопка в шапке)
2. Ждать live-вывода в нижней панели
3. Сессия длится 5–10 минут (или пока не завершатся задачи)

**Можно ли выбрать роль?**
Да! Нажать на стрелочку рядом с кнопкой:
- **Управляющий** (по умолчанию) — обрабатывает задачи во всех отделах параллельно
- **marketing-lead** — только маркетинг
- **dev-lead** — только разработка
- и т.д.

**Авто-режим:**
- Галочка 🤖 **Авто** в шапке — Управляющий сам запускается каждые 10 минут если есть задачи
- Хорош для фонового выполнения (ночью, пока ты спишь)

### 3️⃣ Планёрка (многоотдельные проекты)

**Когда нужна планёрка?**
Когда одна задача требует **нескольких отделов**: лендинг = маркетинг + разработка + юристы.

**Процесс:**
1. Ты создаёшь задачу в дашборде (опционально отмечаешь отделы)
2. Нажимаешь **«▶ Запустить команду»**
3. Управляющий видит что это многоотдельное → запускает планёрку:
   - Пишет Управляющий: *"Для лендинга мне нужны вопросы от маркетинга, разработки и юристов"*
   - Каждый лид отвечает на свой чат дашборда
   - Управляющий собирает вопросы и пишет тебе:
     > **Маркетинг**: Какой бюджет? Целевая аудитория?
     >
     > **Разработка**: На каком хостинге? Нужен ли SEO?
     >
     > **Юристы**: В каких регионах?
4. Ты отвечаешь одним сообщением
5. Управляющий распределяет реальные задачи всем трём отделам + устанавливает зависимости

## Одобрения (Approval Gate)

**Когда задача требует твоего подтверждения:**
- Рискованные операции: `git push`, развёртывание в production, удаление данных
- Задачи с флагом `requires_approval` (обычно P1)
- Межотдельные задачи с бюджетом

**Что ты видишь в Inbox → Approvals:**
```
⚠️ Deploy to production: database migration

[Предпросмотр]
ALTER TABLE products DROP COLUMN old_field;

[Approve] [Reject]
```

**Логика:**
- **Approve** → команда выполняет задачу дальше
- **Reject** → задача возвращается в WIP с комментарием

## Чат с Управляющим

**Как работает:**
1. Левая сторона дашборда → кнопка **🏛 Управляющий** (всегда в топе)
2. Нажал → открывается чат с глобальным собеседником
3. Можешь писать сообщения, задавать уточняющие вопросы

**Примеры общения:**
```
[Ты] Нужен новый лендинг для outdoor billboards.

[Управляющий] Понял. Запускаю планёрку. 
Нужны уточнения от маркетинга, разработки и юристов.

[Управляющий, спустя 5 минут] Вот вопросы:
- Маркетинг: бюджет, ЦА
- Разработка: хостинг, SEO
- Юристы: регионы

[Ты] Бюджет $2k, ЦА = B2B, хостинг AWS, SEO yes, 
Россия + Казахстан.

[Управляющий] Распределяю задачи. 
Маркетолог стартует на копирайт. 
Статус через 2 часа.
```

**Чат сохраняется между сессиями** — Управляющий помнит весь контекст.

## Ошибки и помощь

### "Команда молчит" ⚠️
Хозяйка сессии закончила, но результата нет. Обычно:
- Застрял на approval-gate → откройся Inbox, нажми Approve
- Технический сбой → жди 1 минуту, запусти сессию ещё раз
- Задача слишком сложная → разбей на подзадачи

### "Я вижу красный статус"
Задача перешла в **BLOCKED** (чего-то не хватает). Прочитай комментарий в карточке задачи.

### "Дашборд не открывается"
Проверь:
1. Запущен ли дашборд: `./commands/devboard-start.sh`
2. Правильный ли порт: http://localhost:4999 (или твой кастомный)
3. Нет ли ошибок в терминале: `tail -50 data/dashboard.log`

### Контакт
Глюки и вопросы → chat в дашборде или лог `data/team.log`

## Часто задаваемые вопросы

**Q: Дашборд перезагружу — потеряю ли данные?**
A: Нет. Задачи хранятся в SQLite (`data/tasks.db`). Переживают перезагрузки, если БД не удалишь.

**Q: Могу ли я добавить нового человека в команду?**
A: Это делает HR-роль через диалог планирования. Или вручную — добавить файл роли в `roles/` (но лучше через HR).

**Q: Что если две задачи конфликтуют?**
A: Управляющий видит в приоритете и cross-task зависимости. Спросит уточнение.

**Q: Я могу отредактировать задачу во время сессии?**
A: Лучше не трогать пока работает. После завершения — редактируй спокойно.

**Q: Сколько стоит запустить сессию (API)?**
A: ~0.02$ за Управляющего (haiku). Зависит от объёма задач. Видишь в вкладке **Статистика**.
