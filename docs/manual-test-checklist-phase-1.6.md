# Manual Test Checklist — Phase 1.6

> Исполнитель: owner (нетехнический).
> Цель: убедиться что все изменения Phase 1.6 работают в браузере.

---

## Предусловие

- Чистая БД: `rm data/tasks.db && python dashboard/app.py`
- Чистый localStorage: открыть DevTools → Application → Local Storage → Clear All
- Браузер: открыть `http://localhost:4999`

---

## Чек-лист

### B1 — Нет Acme-следов в ролях

- [ ] **Проверка в терминале:** `grep -ri --include="*.md" 'Acme\|Acme' roles/` → вывод пустой (или только допустимые `pride-tasks` ссылки на инструменты)

---

### B2 — Onboarding: модалка и сохранение контекста компании

- [ ] **Первый запуск (нет company-context.md):** открыть `http://localhost:4999` → UI показывает форму «О компании» (GET `/api/onboarding/company-context` → `{"exists": false}`)
- [ ] **Заполни форму:** введи Название, Описание, Tone of voice, Ценности, Аудиторию → нажми «Сохранить»
- [ ] **Файл создан:** в терминале: `ls data/company-context.md` → файл существует, `cat data/company-context.md` → содержит введённые данные
- [ ] **Повторный GET:** `curl http://localhost:4999/api/onboarding/company-context` → `{"exists": true, "content": "..."}` (content не пустой)
- [ ] **Повторный запуск:** перезапусти дашборд → форма больше не показывается (exists=true)

---

### F1 — Dropdown «Запустить» с выбором роли

- [ ] **Dropdown виден:** на главной странице кнопка запуска содержит стрелку ▼ (dropdown)
- [ ] **Список ролей:** клик на ▼ → popup показывает минимум «Управляющий» и «dev-lead»
- [ ] **Запуск конкретной роли:** выбери «dev-lead» → нажми «Запустить»
- [ ] **Badge появился:** после запуска виден badge «Запущен: dev-lead» (или аналогичное)

---

### F2 — Единая таблица ролей на вкладке Roles

- [ ] **Вкладка Roles:** перейди на вкладку «Роли» → видна одна общая таблица всех ролей
- [ ] **Колонки:** ширина колонок одинакова, нет горизонтального скролла
- [ ] **Все роли:** в таблице присутствуют как dev-роли, так и marketing-роли (если отдел добавлен)

---

### F3 — Default chat: Управляющий по умолчанию

- [ ] **Чистый localStorage:** очисти localStorage (DevTools → Application → Clear)
- [ ] **Открой дашборд:** по умолчанию открыт чат с Управляющим (не с тимлидом)
- [ ] **Заголовок чата:** в шапке чата написано «Чат с Управляющим» или «Управляющий»
- [ ] **Плейсхолдер:** поле ввода содержит «Напиши Управляющему…» (не «Спроси тимлида…»)

---

### F4 — i18n: нет устаревших «тимлид»-фраз

- [ ] **Тишина команды:** запусти дашборд, дождись завершения сессии без результата → в интерфейсе написано «команда молчит» (не «тимлид молчит»)
- [ ] **Заголовок Live:** вкладка Live-вывода содержит «Live-вывод сессии» (не «Live-вывод тимлида»)
- [ ] **Настройки авто-режима:** в Settings → Team → описание авто-режима гласит «Команда сама запускает сессии» (не «Тимлид сам запускает сессии»)

---

### B5 — model_hint: per-task выбор модели

- [ ] **Создай задачу с hint:** через канбан-доску или API создай задачу для dev-lead с `model_hint=haiku`
- [ ] **Запусти dev-lead:** нажми «Запустить» → выбери «dev-lead»
- [ ] **Проверь env:** в логе запуска или в терминале убедись что процесс получил `PRIDE_TEAM_MODEL=haiku`
