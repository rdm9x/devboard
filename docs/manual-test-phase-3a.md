# Manual Test Checklist — Phase 3a (Chat Threads)

**ID Task:** `0dbe6bbe467c`
**Status:** Phase 3a (Chat threads + messages + viewer filters)

Этот документ содержит пошаговую инструкцию для ручного тестирования chat-функциональности.

## Предусловия

- Запущен devboard в dev-режиме: `python dashboard/app.py`
- Браузер открыт на `http://localhost:5000`
- БД инициализирована с миграциями B1 и B2

## Тест 1: Открыть страницу чата

### Шаги

1. Перейди на `/chat`
2. Дождись загрузки страницы (2-3 сек)

### Проверки

- [ ] Страница загружается без ошибок (код 200)
- [ ] Видна левая панель со списком threads
- [ ] В списке есть дефолтный thread **«📌 General»**
- [ ] В центре отображается область для сообщений
- [ ] Нет JavaScript ошибок в DevTools (F12 → Console)

## Тест 2: Создать новый thread

### Шаги

1. На странице `/chat` найди кнопку **«+ New Thread»** (или аналогичную)
2. Введи title: `Test Planning Session 1`
3. (Опционально) Выбери тип: `direct`
4. Нажми **«Create»**

### Проверки

- [ ] Thread создан успешно
- [ ] Новый thread появляется в левом списке
- [ ] Title совпадает: `Test Planning Session 1`
- [ ] Thread отмечен как active (видна иконка активности)
- [ ] Центральная область переключилась на новый thread

## Тест 3: Отправить сообщение в thread

### Шаги

1. Убедись что ты находишься в новом thread'е (из Теста 2)
2. В нижней части найди input-поле для сообщений
3. Введи текст: `Hello, this is the first message`
4. Нажми **«Send»** (или Enter)

### Проверки

- [ ] Сообщение отправляется (нет ошибок)
- [ ] Сообщение появляется в центральной области thread'а
- [ ] Автор сообщения корректен (должен быть `managing-director`)
- [ ] Timestamp видно рядом с сообщением
- [ ] Input-поле очищается после отправки
- [ ] Автоскролл работает (область скроллится к новому сообщению)

## Тест 4: Отправить несколько сообщений

### Шаги

1. В том же thread'е отправь 3 сообщения подряд:
   - `Message 1`
   - `Message 2`
   - `Message 3`

### Проверки

- [ ] Все сообщения появляются в правильном порядке
- [ ] Сообщения отсортированы по времени (старые внизу, новые вверху)
- [ ] Нет дублирования сообщений
- [ ] История сообщений видна при скролле вверх

## Тест 5: Архивировать thread

### Шаги

1. Найди меню/кнопку рядом с thread'ом в списке (например, три точки `⋮`)
2. Выбери опцию **«Archive»** или **«Finish»**
3. (Опционально) Подтверди действие если есть модальное окно

### Проверки

- [ ] Thread архивируется без ошибок
- [ ] Thread пропадает из списка активных threads
- [ ] В списке есть секция **«Archived»** (свёрнутая или раскрытая)
- [ ] Архивированный thread появляется в секции Archived
- [ ] Статус thread'а изменился на `archived` (видно в деталях)

## Тест 6: Просмотр архивированных threads

### Шаги

1. Найди секцию **«Archived»** в левой панели
2. Нажми на неё чтобы раскрыть (если свёрнута)
3. Выбери архивированный thread из Теста 5

### Проверки

- [ ] Архивированный thread открывается
- [ ] История сообщений полностью видна
- [ ] Input-поле для новых сообщений недоступно (disabled или скрыто)
- [ ] Кнопка unarchive/restore (если реализована) видна

## Тест 7: Фильтр viewer=owner (API level)

### Шаги

1. Откройте DevTools → Network tab
2. Создайте новый thread с title: `Viewer Test`
3. Отправьте сообщения от разных авторов (через API вызов):

```bash
# Отправить сообщение от managing-director
curl -X POST http://localhost:5000/api/threads/<thread_id>/messages \
  -H "Content-Type: application/json" \
  -d '{"author": "managing-director", "text": "Visible to owner"}'

# Отправить сообщение от dev-lead
curl -X POST http://localhost:5000/api/threads/<thread_id>/messages \
  -H "Content-Type: application/json" \
  -d '{"author": "dev-lead", "text": "Hidden from owner"}'
```

4. Получите сообщения БЕЗ фильтра:
```bash
curl http://localhost:5000/api/threads/<thread_id>/messages
```

5. Получите сообщения С фильтром viewer=owner:
```bash
curl http://localhost:5000/api/threads/<thread_id>/messages?viewer=owner
```

### Проверки

- [ ] Без фильтра — видны оба сообщения (2 сообщения)
- [ ] С viewer=owner — видно только сообщение от managing-director (1 сообщение)
- [ ] Сообщение от dev-lead исключено из результатов
- [ ] Статус ответа 200 в обоих случаях

## Тест 8: Фильтр viewer=managing-director

### Шаги

1. Используя thread'а из Теста 7, запросите сообщения с viewer=managing-director:

```bash

```

### Проверки

- [ ] Видны ВСЕ сообщения (и от managing-director, и от dev-lead)
- [ ] Фильтр не работает для managing-director (все видны)
- [ ] Количество сообщений = 2
