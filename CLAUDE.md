Этот репозиторий **публичный** (или будет таким). Всё, что попадает в
git-tracked файлы (код, docs/, README, ADR, commit messages, тесты,
шаблоны ролей, фикстуры), **уезжает на GitHub**.

## Что считается личным — НЕ упоминать в git-tracked файлах

- Название компании владельца: **ПРАЙД / PRIDE / pride**
- Имя владельца: **Дмитрий / Dmitry / Rudich**
- Email/контакты: **priderussia.com, Rudich@…**
- Имена клиентов компании: **SOKOLOV, KOTON, Магнит, Спортмастер,
  LC Waikiki**, и любые другие конкретные бренды-заказчики
- Любые конкретные проекты владельца (название лендингов, кампаний),
  если они идут в репо как пример

## Куда МОЖНО писать личное

- `data/` — gitignored (`tasks.db`, `company-context.md`, бэкапы)
- `workspace/` — gitignored (артефакты задач)
- `ПЕРЕДАЧА_СЕССИИ.md` — gitignored
- `MARKETING_PLAN*.md` — gitignored
- `~/.claude/projects/.../memory/` — это auto-memory Claude, **не часть
  репозитория** (живёт в home-папке пользователя)

## Где НЕЛЬЗЯ писать личное

- **commit messages** (видны в `git log` и на GitHub)
- `README.md`, `README.ru.md`, любые `docs/`, `ADR`, `AGENTS.md`,
  `CLAUDE.md` сами по себе
- `roles/*.md` — system-prompts ролей, едут в репо
- Любые `.py`, `.js`, `.html`, `.css`, `.json`, `.yaml` под git
- **Бэкапы БД** — `data/tasks.db.bak.*` НЕ должны быть в tracking
  (БД содержит чаты, где могут быть личные данные)

## Memory tools (`manager_memory_*`) — нюанс

Они пишут в `data/tasks.db` (gitignored). Технически — безопасно.
Но: если когда-нибудь сделают backup БД и положат в репо (как было
с `data/tasks.db.bak.1779727958`) — личное утечёт. Поэтому даже в
manager memory **писать факты про работу**, не про личность владельца.

✅ ОК: «owner предпочитает короткие ответы; не пилить фичи без
обсуждения; задачи требуют статус done через UI, не через MCP»
❌ НЕ ОК: «Дмитрий из ПРАЙД, клиенты SOKOLOV/KOTON, проект про
крышные конструкции»

## Если нашёл существующее упоминание в git-tracked файле

1. **Не делать массовый sed-replace** молча — это большой архитектурный
   refactor (там есть ADR-документы, тесты, фикстуры).
2. Показать список найденного владельцу, дождаться решения по масштабу.
3. Все правки — отдельным коммитом с понятным сообщением.
